This library provides:

## üöÄ Quick Start

### Installation

```bash
npm install @sakhlaqi/ui
```

### Basic Usage

```tsx
import { Button, Input, Card } from '@sakhlaqi/ui/adapters';

function MyComponent() {
  return (
    <Card>
      <Input placeholder="Enter your name" />
      <Button variant="contained" color="primary">
        Submit
      </Button>
    </Card>
  );
}
```

### With Provider Configuration

```tsx
import { ThemeProvider } from '@sakhlaqi/ui';

function App() {
  return (
    <ThemeProvider provider="mui">
      <MyComponent />
    </ThemeProvider>
  );
}
```

## üîë Key Concepts

### Adapters vs Direct Imports

**‚úÖ DO**: Use adapters for flexibility
```tsx
import { Button } from '@sakhlaqi/ui/adapters';
```

**‚ùå DON'T**: Import directly from providers
```tsx
// This tightly couples to MUI
import { Button } from '@sakhlaqi/ui/providers/mui';
```

### Provider Switching

Change providers globally without refactoring code:

```tsx
// Switch from MUI to Ant Design
<ThemeProvider provider="antd">  {/* was "mui" */}
  <App />
</ThemeProvider>
```

### Schema Validation

All JSON configurations are validated at runtime:

```tsx
import { validatePageConfig } from '@sakhlaqi/ui';

const result = validatePageConfig(config);
if (!result.success) {
  console.error('Invalid config:', result.error);
}
```

## üõ†Ô∏è Advanced Features

### JSON-Driven Pages

Render entire pages from JSON:

```tsx
import { PageRenderer } from '@sakhlaqi/ui/renderer';

const pageConfig = {
  metadata: {
    title: 'Dashboard',
    version: '1.0',
  },
  template: {
    type: 'stack',
    props: { spacing: 2 },
    children: [
      {
        type: 'card',
        props: { title: 'Welcome' },
      },
    ],
  },
};

function DashboardPage() {
  return <PageRenderer config={pageConfig} />;
}
```

### Error Handling

Display validation errors inline:

```tsx
import { ValidationErrorDisplay } from '@sakhlaqi/ui/renderer';

const result = validatePageConfig(config);
if (!result.success) {
  return <ValidationErrorDisplay errors={result.error.errors} />;
}
```

## üé® Theming

### Provider-Specific Themes

Each provider respects its own theming system:

```tsx
// MUI theming
import { ThemeProvider, createTheme } from '@mui/material';

const theme = createTheme({
  palette: {
    primary: { main: '#1976d2' },
  },
});

<ThemeProvider theme={theme}>
  <App />
</ThemeProvider>
```

### Consistent API

Despite different providers, the component API remains consistent:

```tsx
// Works with any provider
<Button variant="contained" color="primary">
  Click Me
</Button>
```

## üìù Conventions

### Naming
- **Components**: PascalCase (`Button`, `InputField`)
- **Props**: camelCase (`onClick`, `fullWidth`)
- **Files**: PascalCase for components, kebab-case for docs

### Required Props
Props marked with `*` in documentation are required and will cause TypeScript errors if omitted.

### Optional Props
All other props are optional and have sensible defaults.
